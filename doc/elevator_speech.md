## 또마니또 ##
> 소규모 모임 참여자들을 위해, 
> AI 미션 생성과 실시간 참여 유도 기능을 사용하여,
> 마니또 활동 기간 중 방치되는 문제를 해결하고,
> 시작부터 끝까지 참여하고 싶은 마니또를 만듭니다.
<br>

## 1. 문제 : 마니또는 왜 흐지부지되는가?  ##

| 기존 마니또의 흐름 | 실제 경험 |
|---|---|
| 매칭 → (방치) → 결과 공개 | "뭘 해야 할지 몰라서 그냥 넘어갔다"|
| 시작과 끝만 존재 | "학교에서 했을 땐 마무리가 잘 안 됐다" |
| 중간 과정에 아무 구조 없음 | "흐지부지되는 경향이 있어 친해지기 어려웠다" |

> 기존 마니또는 매칭과 결과 공개에만 집중합니다.
> 활동 기간 동안 참여자는 방치되고, 게임은 재미를 잃습니다.

<br>

## 2. 해결 아이디어 : 참여가 즐거운 마니또 ##
- 또마니또는 중간 **과정을 게임처럼** 만듭니다. 비밀 쪽지, 맞춤형 미션/힌트, AI 결과 리포트 등 또마니또에서 제공하는 다양한 기능은 참여자들이 **간편하게 참여하고, 다 같이 즐길 수 있는** 구조를 제공합니다.

- **AI 맞춤형 미션 —** 뭘 해야 할지 AI가 제안해줍니다
- **비밀 쪽지 —** 익명으로 마니띠에게 메시지 전송
- **의심 대상 투표 —** 내 마니또가 누구인지 추리하고 투표
- **랭킹보드 —** 참여도를 가시화해 동기 부여
- **AI 결과 리포트 —** 나의 활동 스타일을 분석해 마무리

> 시작 → 흥미진진한 과정 → 기억에 남는 마무리


<br>

## 3. 기술 및 구현 ## 
- 또마니또는 프론트엔드인 Flutter에서 **사용자 활동 데이터**를 백엔드인 Spring Boot로 넘기고, 이를 DB에 **실시간으로 저장**한 후 마니또가 종료되는 시점에 **AI call**을 통해 사용자 맞춤형 결과 리포트를 생성, 출력하는 기본적인 파이프라인을 갖습니다.

```
[Flutter 앱]

    ↓ 사용자 활동 데이터

[Spring Boot 백엔드]

    ↓ 실시간 저장

   [DB]

    ↓ 마니또 종료 시점

[텍스트 생성 AI]   → 성향 분석

    ↓ 분석 결과

[이미지 생성 AI]   → 맞춤형 캐릭터 이미지

    ↓

[AI 결과 리포트 화면 출력]
```

| 영역 | 기술 |
|---|---|
| 프론트엔드 | Flutter |
| 백엔드 | Spring Boot |
| AI 파이프라인 | 텍스트 생성 AI → 이미지 생성 AI (2단계) |
| 데이터 | 활동 로그 실시간 누적 → 종료 시 분석 | 


<br>

## 4. MVP 및 배포 ##  

**MVP 핵심 기능**

| 기능 | 설명 |
|---|---|
| 방 생성 & 랜덤 매칭 | 그룹 생성 및 마니또 자동 배정 |
| 비밀 쪽지 | 익명 메시지 전송 |
| AI 결과 리포트 | 활동 데이터 기반 성향 분석 + 캐릭터 이미지 |

**배포 방식**

- 프론트엔드: **안드로이드 APK** 직접 배포 (즉시 설치 가능)
- 백엔드 & DB: **Railway** 클라우드 서버 운영


<br>


## 5. 차별성 ##  

> 또마니또는 '시작과 끝'이 아니라, **그 사이 과정**에 집중하는 마니또 서비스입니다.

| | 기존 마니또 앱/방식 | **또마니또** |
|---|---|---|
| 초점 | 매칭 + 결과 공개 | **활동 과정 전체** |
| 참여 유도 | 없음 | AI 미션 + 투표 + 랭킹 |
| 마무리 | 단순 공개 | AI 분석 리포트 + 캐릭터 |
| 기억에 남는가 | 흐지부지 | 추억이 되는 경험 |

## 6. PMF 및 고객 인터뷰 ##
고객 인터뷰는 기존에 마니또를 경험해본 3인을 대상으로 인터뷰이의 마니또 경험이 어땠는지, 제공하고자 하는 기능에 대해 어떻게 생각하는지를 주요 질문으로 진행되었다.
|  | 전반 의견 | 문제점 | 제품과의 부합성 | 연관 기능 |
| :---: | :--- | :--- | :--- | :--- |
| 마니또를 하는 이유 | 관계 증진을 위함 | 마니또를 하면서 직접적인 관계 개선은 경험하지 못했다 (원래 친하지 않은 경우, 마니또를 하면서 친해지기 어려웠다는 의견 확인됨) | 높음 | 또마니또는 참여 인원 간 전반적인 관계 증진을 위해 의심 투표, 실시간 채팅 등의 상호작용 요소를 포함한다. 모임원들끼리 모이기 어려운 상황에서도 다 같이 대화를 나누는 경험을 통해 가까워질 수 있다. |


## 고객 인터뷰 개요

- 인터뷰 대상: 마니또 경험이 있는 대학생 3인 (A, B, C)
- 인터뷰 방식: 반구조화 인터뷰 (14개 주제, 각 주제별 3인 응답 수집)
- 주요 탐색 질문: 마니또 경험, 불편함, 지속 참여 요인, 재참여 동기

### 고객 인터뷰 요약 (3인)

#### 참여자 A
- 마니또 경험: 중학교 반, 대학 동아리
- 핵심 니즈: 평소 말 안 나누던 사람과 이야기할 계기 원함
- 불만: 마무리가 흐지부지, 해결 시도 없이 그냥 종료됨
- 랭킹에 대해: "친하지 않은 사람 뽑았을 때 스트레스가 될 수 있음"

#### 참여자 B
- 마니또 경험: 대학 동아리 (복수)
- 핵심 니즈: 추리·의심·페이크 치는 과정 자체를 즐김, 경쟁심 있음
- 불만: 선물이 목적이 되어버림, 경제적 부담
- 인사이트: "미션 위주가 선물보다 낫다" → 미션 기능의 직접적 수요 근거
- 랭킹에 대해: "경쟁심 있어서 더 열심히 할 것 같다"

#### 참여자 C
- 마니또 경험: 고등학교 반, 대학 동아리 (복수)
- 핵심 니즈: 흩어진 그룹을 다시 연결하는 계기로 마니또 활용
- 불만: 선물 금액 상한선을 정해도 넘어가는 경우 발생
- 인사이트: "마니띠의 반응이 지속 참여 동기" → 반응 가시화 기능 필요
- 랭킹에 대해: "어떤 사람의 한 번이 다른 사람의 열 번보다 더 큰 정성일 수 있다, 수치 동기부여 안 됨"

<br>
